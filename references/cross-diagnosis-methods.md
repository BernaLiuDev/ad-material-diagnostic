# 交叉诊断方法：多指标交叉 + 二阶导数分析 + 平滑降噪

## 一、多指标交叉诊断方法

单一指标永远只能看到"症状"，只有两个或多个指标的**交叉关系**才能揭示"病因"。以下四个交叉诊断区覆盖了从"用户看到广告"到"产生商业回报"的完整链路。

### 1. 流量引入区交叉诊断 (Hook & Retention)

**核心交叉指标：** 【前3秒完播率】×【5秒完播率 / 整体跳出率】

**解决的业务问题：** 诊断素材开局的"眼球捕获力"与后续内容的"留存衔接度"。

| 3s完播率 | 5s完播率 | 诊断结论 | 行动方向 |
|---------|---------|---------|---------|
| 高 | 高 | Hook 和内容都好，留存优秀 | 放量！不要动内容 |
| 高 | 低 | Hook 吸引了人但内容衔接断裂 | → L2 标签：Hook Deception |
| 低 | 高 | Hook 无力但挺过去的人觉得内容好 | → L2 标签：Slow Burn |
| 低 | 低 | 全面失败 | 考虑归档 |

---

### 2. 意图引导区交叉诊断 (Interest & Action)

**核心交叉指标：** 【整体完播率】×【点击率 CTR】

**解决的业务问题：** 诊断素材是单纯的"好看"，还是具有真实的"商业带货能力"。

| 完播率 | CTR | 诊断结论 | 行动方向 |
|-------|-----|---------|---------|
| 高 | 高 | 内容吸引 + 转化驱动兼备，完美素材 | 全力放量 |
| 高 | 低 | 内容娱乐性强但缺商业转化触发器 | → L2 标签：Entertainment Trap |
| 低 | 高 | 内容不吸引但误打误撞吸引了精准人群 | → L2 标签：Hard Filter |
| 低 | 低 | 既不好看也不带货 | 考虑归档 |

---

### 3. 转化承接区交叉诊断 (Traffic Quality & Matching)

**核心交叉指标：** 【点击率 CTR】×【转化率 CVR】

**解决的业务问题：** 诊断媒体端引来的流量质量，以及广告与落地页的"所见即所得"一致性。这是最容易引起优化师与剪辑师矛盾的地方。

| CTR | CVR | 诊断结论 | 责任归属 |
|-----|-----|---------|---------|
| 高 | 高 | 流量精准，落地页承接到位 | 无问题 |
| 高 | 低 | 点击多但不买——流量泛或广告与落地页不一致 | **优化师责任（定向问题）** 或 **落地页问题** |
| 低 | 高 | 点击少但点的人都买了——素材不够吸引但产品本身ok | **剪辑师责任（素材吸引力不足）** |
| 低 | 低 | 既点不进来也不买 | 全面问题，需同时调整素材和落地页 |

**冲突场景处理：**
当 CTR 高但 CVR 低时，这是优化师和剪辑师最容易互相甩锅的场景。此时需进一步分析：
- 如果 L2 = Hook Deception → 更可能是素材承诺过度，剪辑师需调整
- 如果 L2 正常 → 更可能是定向泛化或落地页不匹配，优化师/产品需调整

---

### 4. 商业收益区交叉诊断 (Cost & ROI)

**核心交叉指标：** 【千次展现成本 CPM】×【单客转化成本 CPA】×【ROAS】

**解决的业务问题：** 这是老板和财务最看重的终局诊断，脱离了内容本身，纯看在竞价环境中的生存状态。

| CPM | CPA | ROAS | 诊断结论 |
|-----|-----|------|---------|
| 低 | 低 | 高 | 效率极高，竞价环境友好 |
| 低 | 高 | 低 | 流量便宜但转化差——内容或产品问题 |
| 高 | 低 | 高 | 流量贵但转化好——产品力强 |
| 高 | 高 | 低 | 流量贵且转化差——全面危机 |

---

## 二、关键指标异动分析（二阶导数分析 Metrics Acceleration）

这是整个诊断体系的时间维度核心。通过一阶和二阶导数，将"静态数据快照"升级为"动态生命周期图谱"。

### 核心理念

```
静态看（没加上时间）：系统只知道 ROI 在掉。
  → 投手觉得："没关系，可能今天大盘不行，我再观察一天。"

动态看（加入微积分）：系统发现加速度不对劲。
  → 得出结论："这是断崖暴雷，再观察一天会多亏5000块。"
```

**关键原则：** 我们不是在写 IF-ELSE 公式，而是在构建**广告放量的派生特征**。把时间（一阶斜率、二阶加速度）作为第四个维度，加入了原本只有平面的漏斗交叉分析中。

### 五个导数驱动的预期结论

#### 结论 1：【指数爆发期 / 黄金吃量期】(Rocket Launch)

**多指标组合表现：**
- 消耗 (Spend) 激增
- ROAS 持续 > 1.2（盈利）
- 一阶导数 > 0，二阶导数 > 0（不仅越来越好，而且增长速度在狂飙）

---

#### 结论 2：【见顶疲劳期 / 拐点微滑】(Growth Bottleneck / Plateau)

**多指标组合表现：**
- 消耗处于高位，ROAS 尚能保本
- 一阶导数 > 0，但**二阶导数 < 0**（还在涨，但是涨幅越来越慢，快到天花板了）
- 或 一阶导数开始 = 0（完全停止增长）

---

#### 结论 3：【平缓衰退期 / 软着陆】(Soft Landing)

**多指标组合表现：**
- 消耗逐渐变少，ROAS 在盈亏线边缘徘徊
- 一阶导数 < 0（在变差）
- 但**二阶导数 > 0**（虽然在变差，但变差的速度正在放缓，趋于一个平稳的低保点）

---

#### 结论 4：【系统停量期 / 僵尸池】(Algorithm Abandonment)

**多指标组合表现：**
- 即便出价很高，日消耗 (Spend) 依然极低，几乎跑不出去钱
- 大盘各项数据没有明显断崖，就是纯纯的曝光量 = 0

---

#### 结论 5：【流量诈尸期 / 二春翻红】(Traffic Renaissance)

**多指标组合表现：**
- 一个沉寂了数周（跑量极低）的素材
- 一阶导数突然 > 0 且成倍数跳跃，转化率异常拉高

**高阶彩蛋：** 这是最难被规则系统捕捉的状态，因为它违反了"下降趋势不可逆"的一般假设。

## 三、平滑波动降误报率

### 问题：为什么需要平滑？

广告数据存在很大的随机噪声：
- **周末效应**：周六日流量模式与工作日完全不同
- **午夜效应**：凌晨数据量极少，波动极大
- **大盘流量波动**：节假日、平台活动等外部因素

如果直接做"点对点"的二阶差分（今天 vs 昨天 vs 前天），二阶导数会疯狂放大这些噪声信号，产生大量误报。

### 解决方案：滑动窗口平滑

**步骤：**

1. **计算移动平均**：使用近 7 天滑动窗口
   ```
   MA(3) = 近 3 天均值
   MA(5) = 近 5 天均值
   ```

2. **基于 MA 序列计算一阶斜率**：
   ```
   一阶导数 ≈ [MA(今天) - MA(3天前)] / 3
   ```

3. **基于一阶斜率序列再求二阶**：
   ```
   二阶导数 ≈ [一阶导(今天) - 一阶导(3天前)] / 3
   ```

4. **替代方案**：线性回归斜率差异 (Trend Line Slope Variance)
   - 对近 7 天数据做线性回归，斜率 = 一阶导的稳健估计
   - 对前 7 天和后 7 天分别做回归，两个斜率的差 ≈ 二阶导
   - 计算量更大但抗噪声能力更强

### 实操选择建议

| 场景 | 推荐方法 |
|------|---------|
| 数据量充足（≥ 14天日级数据） | 线性回归斜率差异（最稳健） |
| 数据量中等（7-14天） | MA(3)/MA(5) 滑动窗口（平衡精度与效率） |
| 数据量不足（< 7天） | 跳过二阶导数分析，仅用一阶趋势 |
| 数据量极少（< 3天） | 跳过所有趋势分析，仅做静态快照 |

## 四、交叉诊断输出格式

在完成三层标签诊断（Phase 1-2）后，补充以下交叉诊断板块：

```
## 交叉诊断补充

### 流量引入区 (Hook & Retention)
- 3s完播率: [值] vs 大盘 [值] → [高/正常/低]
- 5s完播率: [值] vs 大盘 [值] → [高/正常/低]
- 交叉结论: [一句话]

### 意图引导区 (Interest & Action)
- 整体完播率: [值] vs 大盘 [值] → [高/正常/低]
- CTR: [值] vs 大盘 [值] → [高/正常/低]
- 交叉结论: [一句话]

### 转化承接区 (Traffic Quality & Matching)
- CTR: [值] vs 大盘 [值] → [高/正常/低]
- CVR: [值] vs 大盘 [值] → [高/正常/低]
- 交叉结论: [一句话]
- 责任归属: [投手/剪辑师/产品/无问题]

### 商业收益区 (Cost & ROI)
- CPM: [值] vs 大盘 [值]
- CPA: [值] vs 大盘 [值]
- ROAS: [值] vs 盈利线 [值]
- 交叉结论: [一句话]

### 趋势加速度（如有时序数据）
- 核心指标一阶导: [正/负/零] — [在涨/在跌/持平]
- 核心指标二阶导: [正/负/零] — [加速/减速/匀速]
- 导数结论: [一句话，映射到 L1 标签]
```
