# Benchmark 标准与样本阈值拦截规则

## 一、指标分类与 Benchmark 界定方法

不同类型的指标需要不同的 Benchmark 界定方式。

### 1. 核心盈亏指标（ROAS 类）

ROAS 的好坏标准因行业、品类、毛利率差异极大，不存在通用基准。

**界定方式：** 通过交互获取用户指定的上下界限。

| 参数 | 说明 | 默认值（用户未提供时） |
|------|------|---------------------|
| ROAS 盈利线 | 高于此值认为盈利 | 1.2 |
| ROAS 保本线 | 低于此值开始亏损 | 1.0 |
| ROAS 危险线 | 低于此值严重亏损 | 0.8 |

如果 GraphQL 数据中包含用户自定义的 ROAS 阈值，直接使用；否则使用默认值并在报告中标注。

### 2. 比率类指标（CTR / CVR 类）

比率指标有相对稳定的行业参考范围。

**界定方式：** 通用阈值参考历史 **0.8x - 1.2x 大盘均值** 作为正常区间的上下界限。

| 指标 | 低于正常 | 正常范围 | 高于正常 | 极端高位 |
|------|---------|---------|---------|---------|
| CTR | < 0.8x 大盘 | 0.8x - 1.2x 大盘 | 1.2x - 2x 大盘 | > 2x 大盘（前10%） |
| CVR | < 0.8x 大盘 | 0.8x - 1.2x 大盘 | 1.2x - 2x 大盘 | > 2x 大盘（前10%） |
| 3s完播率 | < 0.8x 大盘 | 0.8x - 1.2x 大盘 | > 1.2x 大盘 | > 1.5x 大盘 |
| 全完播率 | < 0.8x 大盘 | 0.8x - 1.2x 大盘 | > 1.2x 大盘 | > 1.5x 大盘 |

**关键前提：** "大盘"必须是同类素材的大盘（同时长分桶、同品类、同平台）。

### 3. 广告放量类指标（Spend / Impression / Click）

放量指标衡量的是"素材被系统喂了多少流量"，越大越好（在核心指标达标的前提下）。

**界定方式：** 使用分位逻辑定义。

| 分位 | 判定 |
|------|------|
| < P50 | 不及格信号（系统不看好） |
| P50 - P75 | 正常水平 |
| P75 - P90 | 良好（系统在给量） |
| > P90 | 爆款级（系统疯狂给量） |

## 二、同类过滤比较（Cohort Filtering）

在进行 Benchmark 比较时，必须先进行同类过滤，确保"苹果跟苹果比"。

### 过滤维度

| 维度 | 说明 | 示例 |
|------|------|------|
| 视频时长分桶 | 不同时长的完播率天然不同 | 15s / 30s / 60s 分开比 |
| 投放平台 | 不同平台的用户行为模式不同 | TikTok / Meta / Google 分开比 |
| 品类 | 不同品类的转化率基准差异大 | 美妆 / 3C / 食品 分开比 |
| 投放地区 | 不同市场的 CPM/CTR 基准不同 | 美国 / 东南亚 / 欧洲 分开比 |

**操作规则：**
- 如果数据中包含时长/平台/品类标签，自动按最细粒度过滤后再比较
- 如果缺少过滤标签，在报告中标注"Benchmark 基于全量大盘数据，未做同类过滤，结论精度可能受限"

## 三、自身环比信号（Self-comparison）

除了与大盘比较，素材的**自身历史变化**也是重要信号。

### 环比监控逻辑

| 场景 | 检测方式 | 信号含义 |
|------|---------|---------|
| Hook 阶段（v_2s / v_3s）环比下降 > 20% | 本周均值 vs 上周均值 | 钩子内容开始疲劳，用户对开头免疫 |
| CTR 环比下降 > 15% | 本周均值 vs 上周均值 | 封面/标题吸引力衰减 |
| CVR 环比下降 > 20% | 本周均值 vs 上周均值 | 落地页或产品匹配度出问题 |
| ROAS 环比下降 > 25% | 本周均值 vs 上周均值 | 综合效率快速恶化 |
| Spend 环比上升但 ROAS 环比下降 | 两个指标反向变动 | 系统在给量但转化在变差，疲劳前兆 |

**输出规则：**
- 即使绝对值仍高于大盘，只要环比出现显著下跌，也必须在报告中标注预警
- 环比信号在 L1 诊断中是辅助判据，不单独构成诊断标签

## 四、样本阈值拦截规则

在进行任何诊断之前，必须先检查数据是否满足最低样本要求。

### 规则 1：样本数量拦截（Spend 数据不足）

| 条件 | 处理方式 |
|------|---------|
| 过去 3 天累计消耗 < 200 美金 | 直接跳过二阶导数分析 |
| 或 单日消耗 < 50 美金 | 加速度没意义，纯噪音 |
| | 直接打标 **【数据样本量过小】** |
| | 不给出具体评价，防止误导 |

### 规则 2：曝光量拦截

| 条件 | 处理方式 |
|------|---------|
| 曝光数据 Impression < 1K | 直接对样本打标 **【数据样本过少】** |
| | 不给出具体体评价，防止误导 |

### 规则 3：极值良性宽容

| 条件 | 处理方式 |
|------|---------|
| 当前值仍远高于大盘均值 × 1.5 | 忽略衰退加速度 |
| | 人家还在高点，跌就跌点 |

**逻辑解释：** 一个 ROAS = 3.0 的素材即使一阶导为负，也不需要恐慌——它从"极好"变成"很好"，和一个 ROAS = 0.9 的素材从"勉强保本"变成"亏损"是完全不同的。

### 规则 4：慎重判罚（多指标叠加）

**只有同时满足以下所有条件**，才打上最严重的【疲劳断崖暴雷】标签：

```
条件 1: 一阶导数 < 0（趋势在下降）
  AND
条件 2: 二阶导数为负且绝对值 > 阈值 a（下降在加速）
  AND
条件 3: 消耗量级属于头部 20%（这是个大号素材，不是小打小闹）
  AND
条件 4: 当前 ROAS 逼近历史 P25 低水位（已经接近危险区）
```

**缺少任何一个条件，都不应该打"断崖暴雷"标签。** 这是最严重的诊断，对应的是"无脑关停"的行动指令，误判的代价极高。

## 五、数据缺失时的降级策略

| 缺失数据 | 影响 | 处理方式 |
|---------|------|---------|
| 无时序数据（只有当天快照） | 无法计算一阶/二阶导数 | 跳过动态分析，仅做静态标签匹配，在报告中标注"缺少时序数据，生命周期判断精度有限" |
| 无内容分镜数据（无完播率） | L2 层不可用 | 跳过 L2 诊断，在报告中标注"无内容留存数据，L2 内容诊断不可用" |
| 无社交互动数据 | L3 层不可用 | 跳过 L3 诊断，在报告中标注"无社交互动数据，L3 情绪诊断不可用" |
| 无大盘 Benchmark 数据 | 无法做相对比较 | 仅做绝对值分析和自身环比，标注"无大盘基准，以下分析基于绝对值判断" |
| 无同类过滤标签 | Benchmark 精度下降 | 使用全量大盘数据，标注"未做同类过滤" |
